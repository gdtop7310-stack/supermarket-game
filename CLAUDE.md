# Claude Code Collaboration Request

このリポジトリは、Codex と Claude Code の両方で開発します。Claude Code で作業するときは、Codex 側の最新変更と衝突しないように、以下のルールで進めてください。

## 目的

Codex がゲームのアップデート、画像生成素材の反映、ロジック修正、GitHub への push を行ったあと、Claude Code 側でも最新状態を正しく把握して続きの作業ができるようにします。

## 作業開始時に必ず行うこと

1. 現在のブランチと状態を確認してください。

   ```bash
   git status
   git branch --show-current
   ```

2. GitHub の最新 `main` を取り込んでください。

   ```bash
   git pull origin main
   ```

3. 直近の変更内容を確認してください。

   ```bash
   git log --oneline -5
   ```

4. ゲームの主要ファイルを確認してください。

   - `game.js`: ゲームロジック、顧客、収穫、棚、レジ、キャッシャー雇用
   - `render3d.js`: Three.js 表示、キャラクター、果物、畑、棚、レジ
   - `index.html`: 画面、HUD、読み込みバージョン
   - `assets/fruit/`: 生成した果物画像アセット

## 現在の重要な状態

- 果物屋ゲームとして制作中です。
- 主人公とお客さんは、顔のない紫・ピンク系のキャラクターです。
- マウス/ドラッグで自由移動できる操作を実装済みです。
- 顧客は果物を取ったあと、レジへ並びます。
- 最初はキャッシャーがいないため、プレイヤー自身がレジに立って会計します。
- お金を払うことでキャッシャーを雇用でき、雇用後は自動でレジ対応します。
- リンゴ、バナナ、ブドウの生成画像を `assets/fruit/` に入れて、ゲーム表示へ反映済みです。
- 直近の反映コミットは `49aee1e Use generated fruit assets in game` です。

## Claude Code への依頼

このプロジェクトで作業するときは、まず GitHub の最新 `main` を pull して、Codex の変更を取り込んでから作業してください。作業後は、何を変えたか、なぜ重要か、壊れる可能性がある場所、どう確認したかを簡潔に残してください。

特に次の点を守ってください。

- Codex が追加したファイルや実装を消さないでください。
- `assets/fruit/` の画像はゲームで使用中なので、削除・リネームする場合は `render3d.js` 側の参照も必ず更新してください。
- ブラウザキャッシュ対策として、`index.html` の `game.js?v=...` と `render3d.js?v=...` は表示に影響する変更のたびに更新してください。
- 変更後はローカルサーバーで `index.html` を開き、ゲーム開始後にコンソールエラーがないか確認してください。
- GitHub に push したら、コミットIDと変更概要をユーザーに伝えてください。

## 推奨確認手順

ローカルで静的サーバーを起動します。

```bash
python -m http.server 8123
```

ブラウザで確認します。

```text
http://127.0.0.1:8123/
```

最低限、次を確認してください。

- スタートボタンでゲームが始まる
- 主人公がマウス/ドラッグ操作で動く
- 畑にリンゴ、バナナ、ブドウが表示される
- 収穫して棚に補充できる
- お客さんが果物を取り、レジに並ぶ
- キャッシャー未雇用時はプレイヤーがレジ対応する
- キャッシャー雇用後は自動会計される
- ブラウザコンソールにエラーが出ていない


このルールにより、Codex と Claude Code が同じ GitHub リポジトリを基準に、交互に安全に作業できます。
